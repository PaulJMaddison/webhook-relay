# Security Policy

## Supported versions

As of `v0.1.0`, security fixes are provided on the latest release series.

## Reporting a vulnerability

Please do **not** disclose security issues publicly before coordinated disclosure.

Use one of these channels:

1. Open a **private GitHub vulnerability advisory** for this repository.
2. Email `security@example.com` (placeholder; replace with your team address).

Please include:

- affected version/commit
- clear reproduction steps
- potential impact
- any suggested remediation

We will acknowledge receipt as quickly as possible and work with you on a fix and disclosure timeline.
